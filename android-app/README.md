# Gadgetbridge Workout Sync

Свободное Android-приложение для надёжной отправки экспортов тренировок Gadgetbridge (`.fit` и `.gpx`) на собственный HTTP-сервер. Лицензия: **GNU GPL v3.0**.

> Скриншоты будут добавлены после первого публичного релиза.

## Возможности

- ручная и приблизительная периодическая синхронизация через WorkManager;
- рекурсивный доступ только на чтение через Storage Access Framework;
- потоковые SHA-256 и multipart-загрузка без чтения файла целиком в память;
- Room-история, повтор временных ошибок и защита клиента от повторов;
- русскоязычный интерфейс и уведомления без Firebase/Google Play Services.

## Сборка и установка

Нужны JDK 17 и Android SDK 35. Выполните `./gradlew assembleDebug`; APK появится в `app/build/outputs/apk/debug/`. Установить его можно командой `adb install app/build/outputs/apk/debug/app-debug.apk`. Тесты и lint: `./gradlew test lint`.

## Настройка

В Gadgetbridge выберите обычную общую папку, например `Documents/Gadgetbridge/`, и экспортируйте туда тренировки. В приложении выберите эту папку, укажите HTTPS URL и Bearer-токен. Android запрещает выбор корня и части `Android/data`; приложение не обходит эти ограничения. Разрешение на папку хранится постоянно, но после его отзыва папку нужно выбрать заново.

Токен хранится отдельно за интерфейсом репозитория настроек и никогда не логируется. Текущая минимальная реализация использует приватное хранилище DataStore приложения, защищённое Android sandbox; резервное копирование базы исключено. Для устройств с повышенными требованиями желательно отключить backup приложения на уровне политики устройства.

## HTTP API

`POST {baseUrl}/api/v1/workouts` получает `Authorization: Bearer …`, `X-File-SHA256`, multipart-поля `file`, `sha256`, `filename` и необязательное `modifiedAt`. Успех — 200/201. Ответ 409 считается успехом только с JSON `{"status":"already_processed"}`. Сервер обязан обеспечивать идемпотентность по SHA-256: соединение может оборваться уже после приёма файла, поэтому одна клиентская БД не гарантирует отсутствие дублей.

Пример: `curl -H 'Authorization: Bearer TOKEN' -H 'X-File-SHA256: HASH' -F file=@activity.fit -F sha256=HASH -F filename=activity.fit https://server.example/api/v1/workouts`.

При отсутствии сети WorkManager ждёт подключения. HTTP 408/425/429/5xx, DNS, timeout и разрыв соединения повторяются с exponential backoff; клиентские ошибки отмечаются постоянными. Файл считается стабильным через 30 секунд после изменения. Если провайдер документов не сообщает время изменения, файл допускается к обработке сразу. Удалённые файлы остаются в истории.

## Почему синхронизация иногда выполняется позже выбранного интервала

Период WorkManager — не точное расписание. Android объединяет фоновые задачи и откладывает их при Doze, экономии энергии и отсутствии сети. Некоторые производители ограничивают фоновые приложения агрессивнее. Сначала проверьте сеть, разрешение на папку и системные ограничения; исключение из оптимизации батареи — только диагностический шаг, обычно оно не требуется.

## Ограничения и безопасность

Release принимает только HTTPS и не включает сетевое логирование. Debug разрешает HTTP лишь для localhost, `127.0.0.1` и эмуляторного `10.0.2.2`. Health-check пока не является обязательным: отсутствие `/api/v1/health` не влияет на сохранение настройки. WorkManager может не запускаться без системной реализации AndroidX Startup на сильно модифицированной прошивке.
